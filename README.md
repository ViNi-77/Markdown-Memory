# Markdown Memory

Markdown Memory is a personal workspace for saving, organizing, editing, previewing, and sharing Markdown documents generated by tools such as Claude, ChatGPT, and Gemini.

Production: <https://markdown-memory.vercel.app>

## What It Does

| Feature          | Description                                                      |
| ---------------- | ---------------------------------------------------------------- |
| Google sign-in   | Auth.js v5 with Google OAuth and Drizzle adapter                 |
| Markdown library | Upload, create, edit, preview, and organize Markdown files       |
| Folder tree      | Group documents in nested folders                                |
| Autosave         | Debounced editor saves with flushes on document and mode changes |
| Public sharing   | Generate read-only share URLs for selected documents             |
| AI handoff       | Copy Markdown and open Claude, ChatGPT, or Gemini                |
| In-app AI        | Gemini-powered summarize/format/custom prompts with BYOK support |

## Privacy And Data

Markdown Memory is not a local-only app. It stores signed-in user data in the configured database so documents can be reopened across devices.

| Data                                            | Storage                             |
| ----------------------------------------------- | ----------------------------------- |
| Markdown documents, folders, and share settings | Neon PostgreSQL                     |
| Auth.js user/account/session data               | Neon PostgreSQL and session cookies |
| Public share tokens                             | Neon PostgreSQL                     |
| User-provided Gemini API key for BYOK           | Browser `localStorage`              |
| Server-side Gemini fallback key                 | Deployment environment variable     |

Do not commit real environment variables, database URLs, OAuth secrets, API keys, `.env.local`, `.vercel`, or local agent/tool metadata.

| ------------------------- | --------------------------------- |
| `/`                       | Auth-protected Markdown workspace |
| `/login`                  | Google sign-in page               |

## Local Setup

Install dependencies:

```bash
npm install
```

Create local environment variables:

```bash
cp .env.example .env.local
```

Required variables:

| Variable             | Required | Notes                                          |
| -------------------- | -------- | ---------------------------------------------- |
| `DATABASE_URL`       | Yes      | Neon PostgreSQL connection string              |
| `AUTH_SECRET`        | Yes      | Generate with `npx auth secret`                |
| `AUTH_GOOGLE_ID`     | Yes      | Google OAuth client ID                         |
| `AUTH_GOOGLE_SECRET` | Yes      | Google OAuth client secret                     |
| `GEMINI_API_KEY`     | No       | Server-side fallback when BYOK is not supplied |

Apply the database schema:

```bash
npm run db:push
```

Run the app:

```bash
npm run dev
```

Open <http://localhost:3000>.

## Google OAuth

For local development, configure a Google OAuth web client with:

Authorized JavaScript origins:

```text
http://localhost:3000
```

Authorized redirect URIs:

```text
http://localhost:3000/api/auth/callback/google
```

For production, add the deployed domain:

Authorized JavaScript origins:

```text
https://markdown-memory.vercel.app
```

Authorized redirect URIs:

```text
https://markdown-memory.vercel.app/api/auth/callback/google
```

## Vercel Deployment

1. Connect `ViNi-77/Markdown-Memory` to Vercel.
2. Add Neon Postgres and set `DATABASE_URL`.
3. Set the required Auth.js and Google OAuth environment variables.
4. Add `GEMINI_API_KEY` only if server-side fallback AI should be available.
5. Push to `main` to deploy.

Recommended production environment variables:

```text
DATABASE_URL
AUTH_SECRET
AUTH_GOOGLE_ID
AUTH_GOOGLE_SECRET
AUTH_URL=https://markdown-memory.vercel.app
NEXTAUTH_URL=https://markdown-memory.vercel.app
GEMINI_API_KEY
```

## Known Notes

- Shared documents are intentionally public to anyone with the generated share URL.
- BYOK Gemini keys are stored in browser `localStorage`; users can clear them from the AI panel by saving an empty key.
- If production Google login fails, verify the production redirect URI in Google Cloud Console.
- If production data access fails, verify Vercel production environment variables and Neon connectivity.
