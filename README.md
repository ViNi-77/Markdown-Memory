# Markdown Memory - 起動・運用マニュアル

このアプリケーションをローカルで起動する方法、および外部（お客さん向け）に公開・運用する際の手順を説明します。

---

## 📋 事前準備

PCに **Node.js** (推奨: v20以上) がインストールされていることを確認してください。

---

## 💻 ローカル環境での起動手順（開発・デバッグ用）

### 1. 依存関係のインストール
プロジェクトのルートディレクトリで以下のコマンドを実行します。
```bash
npm install
```

### 2. 環境変数の設定 (開発用フォールバック)
プロジェクトルートにある `.env` ファイルに、開発用の Gemini API キーを設定できます。
*(※ お客さんに公開する際は、このキーを設定したままにしないか、空にしておきます)*
```env
GEMINI_API_KEY="あなたのAPIキー"
```

### 3. 開発サーバーの起動
Vite 開発サーバーと Express API サーバーを同時に起動します。
```bash
npm run dev
```
起動後、ブラウザで [http://localhost:3000](http://localhost:3000) にアクセスします。

---

## 🚀 外部公開・本番環境でのデプロイ手順

このアプリを顧客向けサーバー（VPSやCloud Run、Vercelなど）にデプロイして一般公開する際の手順です。

### 1. 本番用ビルドの実行
以下のコマンドで、フロントエンドとバックエンドのコードを最適化・コンパイルします。
```bash
npm run build
```
実行すると、`dist/` ディレクトリ配下に静的ファイルおよび本番用サーバーファイル (`server.cjs`) が出力されます。

### 2. 本番環境での起動
ビルドされたファイルを使って、以下のコマンドでサーバーを起動します。
```bash
npm start
```

---

## ⚙️ アプリ公開後の初期設定手順（顧客向け案内）

お客さんがこのアプリにアクセスした際、AI機能を利用するためには以下の初期設定が必要です。

1. **サインイン**:
   * 画面の指示に従い、Googleアカウントでサインインします。
2. **API キーの登録**:
   * 画面左上のサイドバーヘッダーにある**設定ボタン（歯車アイコン ⚙️）**をクリックします。
   * 入力欄にお客さん自身の **Gemini API キー** を入力し、「保存」をクリックします。
   * *(※ 入力されたキーは顧客ブラウザの `localStorage` にのみ保存され、開発者側のサーバーには保存されません。通信はSSL/HTTPS経由で安全に行われます)*
3. **AI分析の開始**:
   * ドキュメントを選択（または新規アップロード）し、右上の「AI ✨」ボタンから分析を開始します。
