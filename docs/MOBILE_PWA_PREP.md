# モバイル / PWA 準備メモ

Markdown Memory の長期ゴールは、PCが手元にない場面でもスマホからMarkdownを確認できる状態にすることです。

このドキュメントでは、いきなりPWA化やネイティブアプリ化へ進む前に満たすべき前提を整理します。

## 現在の位置づけ

現在のUIは、デスクトップの4ペイン構成を主軸にしています。スマホ幅ではフォルダペインを隠し、ファイル一覧のフォルダボタン、本文、詳細を横スクロールと下部ナビで確認する作りです。

PWAの下地として、manifest、PNG/SVGアイコン、オフラインページ、Service Worker は追加済みです。Phase 7 では、ここから「ホーム画面に置いても違和感がないPWA品質」と「スマホでMarkdownを読む体験」の磨き込みへ進みます。現在は、下部ナビの現在地表示、選択中ファイルへの復帰、本文・詳細からの戻る導線を追加し、スマホでもアプリらしく往復できる状態に寄せています。さらに Phase 7D で、ホーム画面追加向けの PNG アイコンと Service Worker のアプリシェル更新を固定しています。Phase 7G では、ホーム画面追加とオフライン導線の実機確認手順を固定しています。

ただし、非公開Markdown本文を端末へ残すオフライン閲覧や、ストア配信相当の本格ネイティブ化はまだ本採用していません。

## 先に守ること

- スマホ幅でも `/demo` が開ける
- ファイル一覧からMarkdown本文へ移動できる
- ファイル一覧から詳細ペインへ移動できる
- Markdown本文が読める
- ログイン、共有、編集まわりのセキュリティ仕様を変えない
- 非公開Markdown本文、API応答、共有ページをService Workerでキャッシュしない

## PWA本採用前の安全判断

Phase 6 では、PWAは「安全な下地」までを採用します。非公開Markdown本文をオフライン閲覧のために保存する仕様は、まだ採用しません。

| 確認項目                 | Phase 6 の判断                                                                                                     |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------ |
| ホーム画面追加時の表示   | `name` は `Markdown Memory`、`short_name` は `MD Memory`。PNG の通常用、maskable 用、Apple touch icon を用意済み。 |
| オフライン時の表示       | ナビゲーション失敗時は `/offline` を表示する。通常利用に戻す導線として `/demo` へのリンクを置く。                  |
| 非公開Markdown本文の扱い | Service Worker ではキャッシュしない。`/api/*`、`/view/*`、`/share/*`、`/login` も対象外。                          |
| オフライン閲覧           | Phase 6 では見送る。採用するなら暗号化、明示同意、削除導線、端末紛失時の説明をセットにする。                       |
| オフライン編集           | 競合解決と同期失敗時の復旧が必要なため、Phase 6 では見送る。                                                       |
| インストール訴求         | 強い訴求は行わない。まずはブラウザでスマホ閲覧が破綻しない状態を優先する。                                         |

将来オフライン閲覧を採用する場合は、ユーザーが明示的に選んだファイルだけを対象にし、ログアウトまたは設定画面から端末キャッシュを削除できることを条件にします。

## 自動確認

Playwright でスマホ幅の最低限の閲覧導線を確認します。

```bash
npm run test:e2e
```

対象は `/demo` です。保存や共有を伴わないため、本番DBやOAuth設定に依存しません。

Phase 6 の前準備として、以下の観点をE2Eで固定します。

- スマホ幅でファイル一覧、本文、詳細へ下部ナビで移動できる
- 下部ナビで現在表示中のペインが分かる
- ファイル一覧から選択中ファイルの本文へ戻れる
- 本文から一覧へ、詳細から本文へ戻れる
- スマホ幅でフォルダ作成、選択、名前変更、削除ができる
- スマホ幅でログイン導線またはサインアウト導線にアクセスできる
- 新規作成したMarkdownをスマホ幅でも編集・プレビューできる
- 詳細ペインに共有、ファイル操作、AI操作が収まっている
- 長いファイル名でも本文ヘッダーが横にはみ出さない
- 長文Markdownをスマホ幅で縦スクロールして読める
- フィードバック導線がスマホ下部から開ける
- manifest に 192/512 PNG、maskable PNG、Apple touch icon を含める
- Service Worker 制御下のオフラインナビゲーションで `/offline` を表示する
- `/`、`/login`、`/api/*`、`/share/*`、`/view/*` を Service Worker cache に入れない
- 長いURL、表、コードブロックがスマホ幅の本文ペイン全体を横に押し出さない

## Phase 6 で採用する情報設計

スマホでは、デスクトップの4ペインをそのまま縮小しません。以下の3段階を基本にします。

| 段階 | 画面         | 主目的                     | 補足                                               |
| ---- | ------------ | -------------------------- | -------------------------------------------------- |
| 1    | ファイル一覧 | 探す、選ぶ、作る           | 初期表示。フォルダ操作とアカウント操作もここに置く |
| 2    | 本文         | 読む、必要に応じて編集する | Phase 6 の主役。選択後は本文へ自然に誘導する       |
| 3    | 詳細         | 共有、AI連携、ダウンロード | 危険操作と補助操作を整理して格納する               |

この設計では、スマホの初期ゴールを「管理」ではなく「確認・閲覧」に置きます。編集、共有、AI操作は残しますが、本文を読む流れを邪魔しない配置にします。

### 次の実装方針

- ファイル選択後、スマホ幅では本文ペインへ移動する（初手として実装済み）
- ファイル一覧からスマホ用のフォルダ管理とアカウント操作へ入れる（実装済み）
- 本文ペインのヘッダーは、長いファイル名でも操作ボタンを押し出さない（実装済み）
- 詳細ペインは、共有、ファイル操作、AI操作をまとまりごとに折りたためる（実装済み）
- 下部ナビは現在表示中の一覧・本文・詳細を示す（実装済み）
- ファイル一覧には選択中ファイルへの復帰導線を出す（実装済み）
- 本文と詳細にはスマホ用の戻る導線を出す（実装済み）
- ホーム画面追加向けの PNG アイコンを manifest と metadata で配信する（実装済み）
- Service Worker の shell cache に PNG アイコンを含める（実装済み）
- Service Worker の runtime cache が安全なアプリシェルに限定されていることをE2Eで確認する（実装済み）
- 長いURL、表、コードブロックが本文ペイン全体を横に押し出さないことをE2Eで確認する（実装済み）
- 削除などの危険操作は、誤タップしやすい下部ナビに置かない
- デスクトップの4ペイン幅調整は維持する

## PWA化前に決めること

| 項目           | 方針                                                                                             |
| -------------- | ------------------------------------------------------------------------------------------------ |
| オフライン閲覧 | 非公開Markdown本文のキャッシュは未採用。導入する場合は暗号化・明示同意・削除導線とセットで判断。 |
| オフライン編集 | 競合解決が必要なため、初期PWAでは見送る候補。                                                    |
| ローカル保存   | BYOKキーとMarkdown本文を同じ扱いにしない。本文保存は慎重に判断。                                 |
| 共有リンク     | PWAでもURLを知る人が読める仕様は変えない。                                                       |
| 通知           | 初期段階では不要。                                                                               |
| ストア配信     | Web PWAの安定後に検討。                                                                          |

## 次のUI候補

スマホ専用UIは、4ペイン横スクロールを土台にしながら、以下の段階式ナビゲーションへ寄せます。

1. ファイル一覧
2. Markdown本文
3. 詳細・共有・AI操作

この形にすると、スマホでは「読む」を優先し、フォルダやアカウントなどの管理操作は必要なときだけ開く設計にできます。
